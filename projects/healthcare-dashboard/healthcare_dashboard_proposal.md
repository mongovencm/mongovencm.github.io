# Conner Mongoven - 3112 Project Proposal

## Title
Athlete Wellness and Performance Monitoring Dashboard

## Vision Statement
This project will expand my professional skills and knowledge by providing hands-on experience developing an interactive application to analyze athlete monitoring data using Python, pandas, and Streamlit. I will strengthen my skills in data analysis, software development, dashboard design, and project documentation. The project will also introduce me to technologies and workflows that are common in healthcare and sports performance analytics.

By building a complete, publicly accessible application and documenting the development process, I will gain experience creating portfolio-quality work that demonstrates both technical and professional skills.

## Motivation
Prior to entering the CSPB program, I earned degrees in Exercise Science and Athletic Training and worked in sports medicine for more than five years. I frequently used athlete and patient monitoring information to help guide clinical decisions and track wellness and recovery.

Although I have transitioned into software development, I am interested in opportunities that combine healthcare, sports medicine, and technology. This project allows me to explore that intersection on a smaller, realistic scale. By creating an athlete wellness dashboard using real-world athlete monitoring data, I can integrate ideas based on my background while building new software development and data analytics skills.

## Specific and Measurable Goals (Learning Objectives)

### Learn the Streamlit Framework
- Develop a functional Streamlit application from scratch demonstrating the ability to create interactive visualizations

### Apply Data Analytics Techniques Using pandas
- Import and process a real-world dataset
- Perform data cleaning and transformations

### Create a User-Friendly Dashboard
- Include interactive visualizations, filtering options, and summaries of key statistics
- Maintain a clean and user-friendly interface

### Deploy a Public Application
- Deploy the application using Streamlit Community Cloud
- Make the project accessible through my GitHub website

### Practice Professional Documentation
- Maintain weekly project updates documenting development decisions, challenges encountered, and lessons learned

## Risks to Project Completion

### New Experience Using Streamlit
- First time using the framework

### Dataset Complexity
- The dataset contains various sizes and structures, increasing complexity

### Scope Management
- The dataset offers many potential directions for analysis, creating risk of expanding beyond the course timeframe

## Mitigation Strategy for Risks

### Streamlit
- Review Streamlit tutorials and examples early in the project
- Maintain reference materials for ongoing use

### Dataset Complexity
- Focus primarily on subjective athlete monitoring data rather than large objective datasets
- Identify and prioritize manageable subsets of the data

### Scope Management
- Establish a minimum viable product (MVP) early
- Prioritize dashboard functionality and deployment before expanding features

## Project Assessments

### Functional Assessments
- Dashboard successfully loads and displays data
- Users can interact with filters and visualizations

### Technical Assessments
- Project uses Python, pandas, and Streamlit
- Application is deployed through Streamlit Community Cloud
- Source code is maintained in a GitHub repository

### Documentation Assessments
- Weekly project updates are completed
- GitHub website contains project summary, updates, and links to deployment and repository
- Final report summarizes goals, implementation, results, and future improvements

### Personal Learning Assessments
- Demonstrate understanding of Streamlit fundamentals
- Demonstrate ability to clean, analyze, and visualize a real-world dataset
- Successfully create a portfolio-quality project related to healthcare and athlete performance monitoring
